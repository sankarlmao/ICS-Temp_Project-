# Ukraine Power Grid Lab: Features and Instructions

## How to Run the Lab
1. Navigate to the project directory: `cd /home/sankar/ukraine_power_grid_lab`
2. Ensure you have Docker and docker-compose installed.
3. Start the lab by running: `docker-compose up --build`

## Architecture and Components
The lab is divided into three isolated network segments simulating a real Operational Technology (OT) environment:
- **Corporate Network (`corporate_net`)**: Contains IT systems.
- **DMZ Network (`dmz_net`)**: The Demilitarized Zone for services bridging IT and OT networks.
- **OT Network (`ot_net`)**: Contains the control systems and physical infrastructure emulators.

## Component Features and Functions
### 1. IT Workstation (`it_workstation`)
- **Network:** `corporate_net`
- **Function:** Represents a vulnerable employee machine in the corporate network.
- **Features:** Runs an Alpine Linux container with SSH enabled. It is pre-configured with weak employee credentials (`employee:password123`). This serves as the entry point for an attacker simulating initial access via compromised SSH credentials. It also includes tools like `nmap` and `curl` for lateral movement. Port 2222 on the host maps to port 22 on this container.

### 2. Firewall (`firewall`)
- **Networks:** `corporate_net`, `dmz_net`
- **Function:** Acts as a reverse proxy, simulating a firewall that allows specific traffic from the corporate network into the DMZ.
- **Features:** Runs Nginx. It listens on port 80 (mapped to 8080 on the host) and proxies HTTP requests to the `hmi_server` located in the DMZ.

### 3. HMI Server (`hmi_server`)
- **Networks:** `dmz_net`, `ot_net`
- **Function:** The Human Machine Interface (HMI) allows operators to monitor and interact with the power grid.
- **Features:** A Python Flask web application.
  - **Vulnerable Login:** The login route (`/`) is intentionally vulnerable to SQL injection (e.g., using `admin` and `' OR 1=1`) and uses weak hardcoded credentials (`admin:admin123`).
  - **Dashboard:** The `/dashboard` route fetches the current grid status from the SCADA server.
  - **Control Actions:** The `/action/<cmd>` route allows an authenticated user to send critical commands to the SCADA server, specifically `turn_off` (simulating a blackout) or `wipe` (simulating a destructive attack).

### 4. SCADA Server (`scada_server`)
- **Network:** `ot_net`
- **Function:** Central control server that bridges the HMI commands to the physical devices (PLCs). It acts as a Modbus Master.
- **Features:** A Python Flask API application.
  - **Status Checking:** The root route (`/`) connects to the PLC via Modbus TCP (port 502) to read holding registers and interpret the status (1 = ONLINE, 0 = OFFLINE - BLACKOUT, 999 = CORRUPTED - UNRECOVERABLE).
  - **Power Control:** The `/turn_off_power` route writes a `0` to the PLC's holding register, effectively shutting off the simulated power.
  - **System Wipe:** The `/wipe_system` route writes `999` to the PLC's holding register, simulating a catastrophic wipe or firmware corruption.

### 5. PLC Device (`plc_device`)
- **Network:** `ot_net`
- **Function:** Programmable Logic Controller. It acts as a Modbus Server (Slave) representing the actual physical power grid infrastructure.
- **Features:** Uses `pymodbus` to expose a Modbus TCP server on port 502. It holds the operational state in its holding registers (initialized to `1` indicating the system is online). The SCADA server interacts with this device to monitor and change the state of the simulated grid.

## Simulated Attack Path
1. **Initial Access:** Attacker connects to the IT workstation via SSH (`ssh -p 2222 employee@localhost` with password `password123`).
2. **Reconnaissance:** Attacker discovers the firewall service in the corporate network.
3. **Lateral Movement:** Attacker accesses the HMI web interface through the firewall proxy.
4. **Exploitation:** Attacker bypasses the HMI authentication using SQL injection (`' OR 1=1`) or weak credentials.
5. **Impact:** Attacker uses the HMI dashboard to send malicious commands (`turn_off` or `wipe`) to the SCADA server, which forwards Modbus commands to the PLC, causing a simulated power outage or system destruction.
